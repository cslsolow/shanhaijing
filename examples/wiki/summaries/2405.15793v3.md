# SWE-agent: Agent-Computer Interfaces Enable Automated Software Engineering

LM agents 在数字环境中的性能很大程度上取决于它们使用的**接口设计**，而非底层模型本身。本文提出 Agent-Computer Interface（ACI）概念，并基于此构建 SWE-agent，在 SWE-bench 上以 GPT-4 Turbo 达到 12.47% pass@1，大幅超越此前 3.8% 的 SOTA。

## 核心观点

- **ACI（Agent-Computer Interface）**：LM agent 是一类新型终端用户，有别于人类，需要专门设计的接口。ACI 是 LM agent 与计算机之间的抽象层，定义可用命令集和环境反馈格式。
- **四条 ACI 设计原则**：
  1. 命令应简单易理解，文档简洁，减少选项数量
  2. 命令应紧凑高效，单步操作覆盖高阶任务
  3. 环境反馈应信息丰富但精炼，避免无关上下文
  4. 内置 guardrail 防止错误传播，帮助 agent 快速恢复
- **SWE-agent 组件**：search/navigation（find_file, search_file, search_dir）、file viewer（带行号窗口）、file editor（带语法检查 guardrail）、context manager
- **可移植性**：ACI 设计对模型无关；Claude 3 Opus 使用同一 ACI 可解决 10.5% 任务

## 方法

基于 Linux shell 构建，在其上叠加 ACI 层。每步 agent 生成 thought + command，接收执行反馈（ReAct 框架）。通过人工检查 agent 行为 + grid search 优化 ACI 配置。

## 主要结果

| 设置 | SWE-bench 解决率 |
|------|-----------------|
| 非交互式检索增强 LM（prior SOTA） | 3.8% |
| SWE-agent (GPT-4 Turbo) | 12.47% |
| SWE-agent (Claude 3 Opus) | 10.5% |
| SWE-agent on SWE-bench Lite | +10.7pp vs baseline shell |
